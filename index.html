Project Overview

The MOSDAC AI Help Bot is a comprehensive RAG (Retrieval Augmented Generation) system designed to help users find information from the MOSDAC (Meteorological & Oceanographic Satellite Data Archival Centre) website.

Key Features

  • ü§ñ AI-powered question answering with citations
  • üåê Comprehensive web scraping of MOSDAC content
  • üìä Automatic data ingestion every 48 hours
  • üîç Semantic search with multi-modal embeddings
  • üí¨ Natural language chat interface
  • üöÄ REST API for frontend integration
  • üìà Real-time system monitoring

Problem Statement

MOSDAC (www.mosdac.gov.in) hosts vast amounts of satellite data and services, but users often struggle to find specific information due to navigation complexity and mixed content. This AI help bot provides instant, accurate responses based on the website's content.

Python Client Example

import requests import json class MOSDACClient: def __init__(self, base_url="http://localhost:8000", api_key=None): self.base_url = base_url self.api_key = api_key self.session = requests.Session() if api_key: self.session.headers.update({"X-API-Key": api_key}) def chat(self, query, session_id="default"): response = self.session.post( f"{self.base_url}/api/v1/chat", json={"query": query, "session_id": session_id} ) return response.json() def status(self): response = self.session.get(f"{self.base_url}/api/v1/status") return response.json() # Usage client = MOSDACClient(api_key="your_api_key") response = client.chat("What satellite data is available?") print(json.dumps(response, indent=2))

Troubleshooting Common Issues

Problem: LLM not responding

Solution: Check your LLM configuration in .env file

# For API mode: Ensure GEMINI_API_KEY is valid # For Ollama mode: Ensure Ollama is running on localhost:11434 # Test: python scripts/setup_llm.py --test

Problem: Scraping fails

Solution: Check internet connection and website availability

# Test connectivity: curl -I https://mosdac.gov.in # If website is down, scraping will fail # Check: https://mosdac.gov.in status

Problem: API not starting

Solution: Check port availability and dependencies

# Check if port 8000 is available: netstat -tuln | grep :8000 # If port busy, change API_PORT in .env API_PORT=8001 # Reinstall dependencies if needed: pip install -r requirements.txt

Problem: Memory issues

Solution: The system uses ~5GB RAM during processing

# For lower memory systems: # 1. Reduce chunk size in enhanced_chunker.py # 2. Use smaller embedding models # 3. Process in smaller batches # Minimum recommended: 8GB RAM # Ideal: 16GB+ RAM
